Date: 2026-07-22
Source: `apps/web/public/media/red-masai/concept/`

> for private owner demonstration and excluded from Git history. Public
> publication requires owner confirmation, ownership verification,
> subject-consent review and replacement or approval of any copyrighted screen,
> poster, broadcast or audio content.

## Scope and status

This inventory began as a planning and rights-review document and now drives the tracked Milestone 4C placement catalogue. The 22 original media assets remain unchanged, unoptimized, ignored, unstaged and uncommitted. The source folder's `README.md` was reviewed before this inventory was prepared. The tracked catalogue controls local placement but is not evidence of publication clearance.

The current files are suitable for a private owner demonstration as review material. They must not be treated as production-approved media. A public concept preview is recommended only where the table says **Yes**, and only while the concept-preview notice remains enabled. **Approval required** means written owner confirmation of copyright/publication rights plus consent from every recognizable person; it does not imply that consent already exists.

Where food, beverages, decorations, bath styling or games appear, captions and nearby copy must describe an example setup or atmosphere, never an automatic inclusion. Visible screens, posters and branded packaging should be cropped or de-emphasized. Final public media should also receive an owner-approved rights/channel record.

### Safety key

- **Private:** safe to show in a controlled owner demonstration for review.
- **Public:** safe for a public concept preview in the proposed crop, subject to the general owner/copyright confirmation applying to every photograph.
- **Approval:** asset has an additional, specific approval or clearance gate. It should remain out of a public preview until that gate is cleared.

## Milestone 4C implementation requirements

Implemented for private owner review:

- Keep the concept-preview notice enabled.
- Preserve the current safe media placeholders whenever an asset is absent, withheld or fails to load.
- Supply explicit `width`/`height` or CSS `aspect-ratio` for every image and the video; lazy-load below-the-fold media.
- Keep the Create video muted by default and expose controls. Do not autoplay when `prefers-reduced-motion: reduce` is active; a poster and manual play must remain available.
- Use art-directed crops to exclude or de-emphasize screens, posters, packaging and other protected content rather than modifying the originals.
- Keep approval-gated assets out of any public concept preview until the rights record is complete.
- Do not stage or commit the media files until the media-rights review is approved.

The catalogue lives at `apps/web/src/lib/red-masai-media.ts`; the reusable resolver lives at `apps/web/src/components/sections/concept-media.tsx`. Only literal `NEXT_PUBLIC_RED_MASAI_DEMO_MEDIA=true` enables local media. Missing, false or malformed values preserve the complete placeholder layout. Individual image/video load errors return to the same placeholders without displaying broken media.

Because ignored media is not protected by GitHub and Codespaces is not a permanent archive, retain the original ZIP plus a private cloud backup and a local laptop or phone copy. Do not upload these automatically.
